## Why We Built This

I love Next.js's file-based routing. The idea of just creating a folder and a file and having a route automatically appear ‚ÄĒ no configuration, no imports, no `<Route>` declarations ‚ÄĒ is genuinely brilliant.

**But I don't want to use Next.js.**

I want React. I want Vite. I want the speed, the freedom, and the simplicity of a plain React project ‚ÄĒ without being forced into a full framework just to get one feature I love.

So I built **upcoming.js** ‚ÄĒ Next.js-style file routing, for React + Vite. Nothing more, nothing less.

---

## What is upcoming.js?

**upcoming.js** is a Vite plugin that watches your folder structure and automatically generates routes for React Router ‚ÄĒ no setup, no manual route declarations, no `<BrowserRouter>` wiring.

You create folders. It creates routes.

```
src/
‚Ēú‚ĒÄ‚ĒÄ page.jsx          ‚Üí   /
‚Ēú‚ĒÄ‚ĒÄ about/
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ page.jsx      ‚Üí   /about
‚Ēú‚ĒÄ‚ĒÄ blog/
‚Ēā   ‚Ēú‚ĒÄ‚ĒÄ page.jsx      ‚Üí   /blog
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ [id]/
‚Ēā       ‚ĒĒ‚ĒÄ‚ĒÄ page.jsx  ‚Üí   /blog/:id
‚ĒĒ‚ĒÄ‚ĒÄ notfound.jsx      ‚Üí   404
```

That's it. That's the whole idea.

---

## Features

* ūüďĀ **Folder-based routing** ‚ÄĒ your file structure is your route structure
* ‚ö° **Vite plugin** ‚ÄĒ zero config, plugs directly into your existing Vite setup
* ūüĒĄ **Live route updates** ‚ÄĒ add or delete a `page.jsx` and the routes update instantly, no server restart needed
* ūüĒÄ **Dynamic routes** ‚ÄĒ `[param]` folders become `:param` in the URL
* ūüóāÔłŹ **Route groups** ‚ÄĒ `(group)` folders organize your code without affecting the URL
* ūüôą **Private folders** ‚ÄĒ `_folder` folders are completely ignored by the router
* ūüöę **404 pages** ‚ÄĒ just create `notfound.jsx` at the root of your routes folder
* ūüí§ **Lazy loading** ‚ÄĒ every page is automatically code-split and lazy loaded
* ūü§Ě **React Router powered** ‚ÄĒ all navigation, params, and hooks work exactly as you already know

## Installation

```bash
npm install upcoming.js
```

> **upcoming.js** will automatically install `react-router` as a dependency.
> Make sure you have `react`, `react-dom`, and `vite` already in your project.

---

## Setup

### 1. Add the plugin to `vite.config.js`

```js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import upcoming from 'upcoming.js'

export default defineConfig({
  plugins: [react(), upcoming()],
  resolve: {
    dedupe: ['react', 'react-dom', 'react-router'] // (optional) for better performance
  },
  optimizeDeps: {
    include: ['react-router'] // (optional) for better performance
  }
})
```

> The `dedupe` line prevents duplicate React instances ‚ÄĒ always include it.

### 2. Replace your `src/main.jsx`

```jsx
import ReactDOM from 'react-dom/client'
import { UpcomingRouter } from 'upcoming.js/runtime'

ReactDOM.createRoot(document.getElementById('root')).render(
  <UpcomingRouter />
)
```

Done. No `App.jsx`, no `<BrowserRouter>`, no route declarations. Just run `npm run dev`.

## Folder & File Conventions

### `page.jsx` ‚ÄĒ defines a route

Every route is a `page.jsx` (or `page.js`) file. The folder path becomes the URL.

```
src/page.jsx                   ‚Üí  /
src/about/page.jsx             ‚Üí  /about
src/settings/account/page.jsx  ‚Üí  /settings/account
```

---

### `[param]` ‚ÄĒ dynamic segments

Wrap a folder name in square brackets to make it a dynamic route parameter.

```
src/blog/[id]/page.jsx         ‚Üí  /blog/:id
src/shop/[category]/page.jsx   ‚Üí  /shop/:category
src/user/[id]/posts/page.jsx   ‚Üí  /user/:id/posts
```

Accessing the param works exactly like React Router:

```jsx
import { useParams } from 'react-router'

export default function BlogPost() {
  const { id } = useParams()
  return <h1>Post ID: {id}</h1>
}
```

---

### `(group)` ‚ÄĒ route groups

Wrap a folder name in parentheses to group routes without affecting the URL. Great for organizing by feature or layout without polluting the URL.

```
src/(auth)/login/page.jsx      ‚Üí  /login
src/(auth)/signup/page.jsx     ‚Üí  /signup
src/(dashboard)/home/page.jsx  ‚Üí  /home
```

---

### `_folder` ‚ÄĒ private folders

Prefix a folder name with `_` to exclude it from routing entirely. Use this for shared components, utilities, or anything that lives inside your routes folder but is not a route.

```
src/_components/Navbar.jsx     ‚Üí  ignored
src/_utils/helpers.js          ‚Üí  ignored
src/about/page.jsx             ‚Üí  /about  ‚úď
```

---

### `notfound.jsx` ‚ÄĒ 404 page

Place a `notfound.jsx` at the root of your routes folder to handle unmatched routes.

```
src/notfound.jsx               ‚Üí  * (catches all unmatched routes)
```

```jsx
export default function NotFound() {
  return <h1>404 ‚ÄĒ Page not found</h1>
}
```

## Navigation

Navigation works exactly like React Router. **upcoming.js** is just an API on top of it ‚ÄĒ all hooks and components are the same.

```jsx
import { Link, useNavigate, useParams, useLocation } from 'react-router'

// Link component
<Link to="/about">About</Link>

// Programmatic navigation
const navigate = useNavigate()
navigate('/blog/123')

// Route params
const { id } = useParams()

// Current location
const location = useLocation()
```

## Configuration Options

```js
upcoming({
  routesDir: 'src'  // the folder to scan for routes ‚ÄĒ default is 'src'
})
```

If you use a feature-based architecture or want routes in a different folder:

```js
upcoming({
  routesDir: 'src/pages'
})
```

## Custom Loading State

You can pass a custom loading component that shows while lazy-loaded pages are being fetched:

```jsx
<UpcomingRouter loading={<div>Loading...</div>} />
```
